## transformers.js v4 (@huggingface/transformers 4.2.0)

```js
// worker.js — 공식 예제(whisper-word-timestamps) 패턴
import { pipeline } from '@huggingface/transformers'

const PER_DEVICE_CONFIG = {
  webgpu: { dtype: { encoder_model: 'fp16', decoder_model_merged: 'q4' }, device: 'webgpu' },
  wasm: { dtype: 'q8', device: 'wasm' },
}
// 모델: onnx-community/whisper-large-v3-turbo (다국어, ko 지원) / 경량: onnx-community/whisper-base
const transcriber = await pipeline('automatic-speech-recognition', MODEL_ID, {
  ...PER_DEVICE_CONFIG[device],
  progress_callback,  // 다운로드 진행률: {status:'progress', file, loaded, total, progress}
})
// WebGPU 워밍업 (셰이더 컴파일): await transcriber(new Float32Array(16_000), { language: 'ko' })
// 전사 (긴 오디오):
const out = await transcriber(float32Audio16k, {
  language: 'ko', chunk_length_s: 30, stride_length_s: 5, return_timestamps: true,
})
// out.chunks = [{ text, timestamp: [startSec, endSec] }]
```

- WebGPU 감지: `navigator.gpu && await navigator.gpu.requestAdapter() !== null` (adapter null 체크 필수)
- 입력: **16kHz mono Float32Array**
- 모델·WASM 캐싱: `env.useBrowserCache` 기본 true → Cache API 자동 (설정 불필요)
- turbo 권장 dtype: encoder fp16 + decoder q4 (fp32 encoder는 별도 .onnx_data 대용량). WASM 폴백은 base 이하만
- `language: 'ko'` 또는 `'korean'` 둘 다 허용

## Groq 전사 API (CORS 실측 확인: `access-control-allow-origin: *`)

```js
const form = new FormData()
form.append('file', blob, 'recording.webm')  // 확장자는 blob.type과 일치시킬 것
form.append('model', 'whisper-large-v3-turbo')
form.append('language', 'ko')
form.append('response_format', 'verbose_json')
const res = await fetch('https://api.groq.com/openai/v1/audio/transcriptions', {
  method: 'POST',
  headers: { Authorization: `Bearer ${apiKey}` },  // Content-Type 직접 지정 금지 (boundary 자동)
  body: form,
})
// 429: res.headers.get('retry-after') (초). 응답: data.text, data.segments = [{start, end, text, ...}]
```

- 무료 한도(2026-07): 20 RPM / 2,000 RPD / 시간당 오디오 7,200초 / 일 28,800초 / **파일 25MB**
- 지원 포맷: flac, mp3, mp4, mpeg, mpga, m4a, ogg, wav, **webm**
- verbose_json 최상위 래퍼 키는 배포 후 실응답 로깅으로 확정할 것 (segments 배열은 확정)

## 오디오 디코딩 (whisper-web 레퍼런스 패턴)

```js
const ctx = new AudioContext({ sampleRate: 16_000 })  // 디코딩 결과가 16kHz로 자동 리샘플 (1h ≈ 230MB)
const decoded = await ctx.decodeAudioData(arrayBuffer) // 완전한 파일만 가능 — 부분 조각 불가
let audio
if (decoded.numberOfChannels === 2) {
  const [l, r] = [decoded.getChannelData(0), decoded.getChannelData(1)]
  audio = new Float32Array(l.length)
  for (let i = 0; i < decoded.length; ++i) audio[i] = (Math.SQRT2 * (l[i] + r[i])) / 2
} else audio = decoded.getChannelData(0)
// 방어: decoded.sampleRate !== 16000이면 OfflineAudioContext(1, Math.ceil(decoded.duration*16000), 16000)로 재리샘플
```

- MediaRecorder timeslice 청크는 단독 디코딩 불가 → 전체 결합 Blob으로만
- MediaRecorder webm은 duration 메타데이터가 NaN/Infinity일 수 있음 (PCM 복원은 OK)
- Safari의 webm/opus decodeAudioData 호환은 미검증 → try/catch + Groq 경로 안내

- 파이프라인: pyannote segmentation-3.0 ONNX(~6.6MB) + 3D-Speaker CAM++ 임베딩(~28MB) + agglomerative clustering
- 공식 브라우저 데모: HF Spaces `k2-fsa/web-assembly-speaker-diarization-sherpa-onnx`
- 빌드: `build-wasm-simd-speaker-diarization.sh` (INITIAL_MEMORY 512MB, ALLOW_MEMORY_GROWTH)
- 화자 수: 자동(코사인 임계값 ~0.5) 또는 수동 지정
- 병합: 전사 세그먼트 시간 구간과 화자 타임라인의 최대 교집합 화자 배정 (WhisperX 패턴)
- 주의: transformers.js의 pyannote-segmentation-3.0 단독은 **세그멘테이션만**(전역 화자 라벨 불가) — sherpa-onnx 완전 파이프라인 필요. 임베딩 모델은 영어 학습이라 한국어 화자 정확도 실측 필요
- Gemini 화자분리 경로는 탈락: 1시간급 라벨 일관성 문제 + Files API(>20MB) 브라우저 CORS 차단

## Gemini generateContent (2026-07-07 재검증 — Plan 4)

- **모델: `gemini-3.5-flash`** (Stable, 무료 티어). 저가 대안 `gemini-3.1-flash-lite`. ~~gemini-2.0-flash~~ **종료됨(Shut down)** — 사용 금지. 2.5-flash는 preview로 재분류
- 엔드포인트: `POST https://generativelanguage.googleapis.com/v1beta/models/gemini-3.5-flash:generateContent`
- 헤더: `x-goog-api-key: <KEY>` + `Content-Type: application/json` (CORS 프리플라이트 실측 허용 — allow-headers에 x-goog-api-key 포함)
- 요청: `{ "contents": [{ "role": "user", "parts": [{ "text": "..." }] }], "systemInstruction": { "parts": { "text": "..." } }, "generationConfig": { "temperature": 0.3 } }`
- 응답 텍스트: `data.candidates?.[0]?.content?.parts` → **`parts.map(p => p.text).join('')`** (여러 조각 가능). candidates 부재 시 promptFeedback(안전필터) 확인
- **에러 시맨틱 (실측)**: 무효 키 = **HTTP 400** `error.details[].reason === 'API_KEY_INVALID'` (401 아님!), 키 없음/권한 = 403 PERMISSION_DENIED, 레이트리밋 = 429 RESOURCE_EXHAUSTED — 대기값은 헤더가 아니라 **바디 `error.details[]`의 RetryInfo `retryDelay`("30s" 형태)**
- 무료 티어 한도는 공식 정적 게시 중단 — AI Studio rate-limit 페이지에서 프로젝트별 확인 (앱 안내문에 링크)

## Whisper 어휘 힌트 (2026-07-10 검증 — 결론: 미지원)

transformers.js 3.8.1은 Whisper용 `initial_prompt`/`prompt_ids`/`get_prompt_ids`/`condition_on_prev_tokens` 1급 API가 **없음**(소스 실확인 — generation_whisper.js의 prompt_ids는 타입 필드 잔재, models.js에서 주석 처리). `decoder_input_ids`로 SOT 프리픽스를 대체하는 우회는 가능하나 `_strip_prompt` 미이식이라 **프롬프트 단어가 전사문에 그대로 누출**되고 청크마다 반복 강제됨 → 신뢰 불가. 이슈 huggingface/transformers.js#923 open.
→ **대안(채택): 후처리 find-replace 보정 사전.** 등록 어휘를 전사 출력에 매번 적용(단어경계·길이 내림차순·대소문자 처리). 원문 보관 후 재적용 가능. Whisper 재추론 없이 즉시 교정.
