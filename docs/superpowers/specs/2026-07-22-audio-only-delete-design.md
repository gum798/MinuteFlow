# 오디오만 삭제 (Audio-only delete)

2026-07-22 · 승인됨

## 목적

오디오가 저장 공간의 대부분을 차지한다(회의당 수십 MB~GB). 전사·화자·요약이 끝난 회의는
오디오를 지워 공간을 회수하되, 회의록 자체(전사·화자 라벨·요약·제목)는 그대로 유지한다.

## 동작

- 회의 화면의 "오디오 다운로드" 옆에 **"오디오만 삭제"** 버튼 (오디오가 있을 때만 표시).
- 누르면 즉시 화면에서 오디오 관련 버튼(다운로드·오디오만 삭제·고품질 재전사·화자 구분)이
  사라지고, 실행취소 토스트(5초)가 뜬다 — 기존 회의 삭제와 동일한 패턴.
- 실행취소: 아무것도 삭제되지 않은 상태로 복귀(실제 삭제는 토스트 만료 시점).
- 만료: 그룹(분할) 회의면 **전체 부**의 오디오 청크를 하드 삭제, 각 회의의 `audioBytes`를
  0으로 갱신(홈 저장 공간 바에 즉시 반영). 세그먼트·요약·회의 행은 유지.
- 작업(재전사·화자 구분·요약) 진행 중엔 버튼 비활성화 — 진행 중 작업이 오디오를 잃지 않게.

## 구현

- `store/meetings.ts`: `deleteMeetingAudio(ids: string[])` — 한 트랜잭션으로
  audioChunks 삭제 + `meetings.audioBytes = 0`.
- `Meeting.tsx`: 버튼 + `audioAvailable` 끄기 + `showUndoToast`(onUndo: 상태 복귀,
  onExpire: `deleteMeetingAudio(group ids)`). "오디오 다운로드" 버튼도 `audioAvailable`로 가드.

## 알려진 한계

토스트 5초 안에 탭을 닫으면 삭제가 실행되지 않고 오디오가 남는다(안전한 방향의 실패 —
다음 방문에 다시 삭제하면 됨). 마커+청소 스윕은 YAGNI로 보류.

## 테스트

- 스토어: 청크 삭제·audioBytes 0·세그먼트/요약 보존·`getMeetingAudio` null·여러 부 일괄.
- UI: 클릭 → 오디오 버튼 숨김+토스트, 실행취소 → 복귀(청크 무손상), 만료 → 실제 삭제.
